### 🚀 طريقة الاستخدام

1. **التحضير**: احفظ نسخة احتياطية من ملفاتك المهمة
2. **التشغيل**: افتح التطبيق (اختر العربية أو الإنجليزية)
3. **الموافقة**: اقرأ إخلاء المسؤولية واضغط على "موافق"
4. **إضافة الملفات**:
   - اسحب وأفلت الملفات في المنطقة المخصصة
   - أو اضغط على زر "إضافة ملفات"
5. **الانتظار**: سيعالج التطبيق الملفات تلقائياً في الخلفية
6. **الحفظ**:
   - اضغط على أي ملف لحفظه منفرداً
   - أو استخدم زر "حفظ الكل" لحفظ جميع الملفات الناجحة

#### ⚠️ تنبيه مهم بعد إزالة حماية VBA

1. افتح الملف المعالج في Access
2. ادخل إلى محرر VBA (Alt + F11)
3. قد تظهر رسالة خطأ "Invalid Key" أو ما شابه - اضغط OK
4. احفظ الملف (Ctrl + S) ثم أغلقه
5. أعد فتح الملف - ستجد محرر VBA مفتوحاً بدون كلمة مرور

### 🚀 How to Use

1. **Preparation**: Backup your important files first
2. **Launch**: Open the application (choose Arabic or English)
3. **Accept**: Read the disclaimer and check "I agree"
4. **Add files**:
   - Drag and drop files into the drop area
   - Or click "Add Access files" button
5. **Wait**: Application processes files automatically in background
6. **Save**:
   - Click any file to save it individually
   - Or use "Save all files" button

#### ⚠️ Important Note After VBA Protection Removal

1. Open the processed file in Access
2. Enter VBA editor (Alt + F11)
3. "Invalid Key" or similar error message may appear - click OK
4. Save the file (Ctrl + S) and close it
5. Reopen the file - VBA editor will open without password

<div id="Eremoved-protections"></div>

### 🔒 Types of Protections Removed

*   **Database Password**: Opens password-protected databases.
*   **VBA Protection**: Removes the password for the VBA editor.
*   **User-Level Security**: Disables user permissions in older (MDB) databases.
*   **Protected Startup Options**: Allows modification of database startup options.

---

<div id="Eprocessing-methods"></div>

### ⚙️ Different Processing Methods

*   `process_accdb_format()`: Used for modern Access files (.accdb, .accde, .accdt).
*   `process_mdb_format()`: Used for legacy Access files (.mdb, .mde).

---

<div id="Eimportant-notes"></div>

### ⚠️ Important Notes

*   **Full Encryption**: If the database is fully encrypted, it might not be possible to open it after processing.
*   **Executable Files**: ACCDE and MDE are executable files, and protection removal might be limited (e.g., original VBA source code might not be restorable).
*   **Backup**: It is crucial to create a backup of your files before starting the processing.
*   **Compatibility**: Different Access versions may handle protection and removal methods differently, which might affect the tool's effectiveness in some cases.
